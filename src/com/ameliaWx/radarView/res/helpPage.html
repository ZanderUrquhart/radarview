  • SPC Watch Parallelograms now display with storm-scale warnings.
  • City markers added to basemap. Press C to toggle visibility.
  • Config settings can now be edited in the program with CTRL+H, CTRL+R, and CTRL+T.
  • Performance improvements, primarily in initialization time and memory consumption.

Precipitation Types

Internally, the program diagnoses 12 different precipitation types using a revised version of the Bourgouin technique (Birk et al. 2021), with additional freezing rain and snow subtypes added by me. Other algorithms will be added at a later time, such as the original Bourgouin technique developed in 2000. Since viewing all 12 types can be confusing, RadarView is set by default to simplify this into a three- or four-type view. The simplifications are as follows:

Twelve-type

[CTRL+4]		 Three-type
[CTRL+3]

Freezing Rain (Elevated) Freezing Rain Wintry Mix
Freezing Rain (Surface) Freezing Rain Wintry Mix
Mixed Freezing Rain and Ice Pellets Freezing Rain Wintry Mix
Ice Pellets Ice Pellets Wintry Mix
Mixed Ice Pellets and Snow Ice Pellets Wintry Mix
Mixed Rain and Ice Pellets Ice Pellets Wintry Mix
Mixed Rain and Snow Snow Wintry Mix
Mixed Freezing Rain and Snow Freezing Rain Wintry Mix
Wet Snow Snow Snow
Dry Snow Snow Snow

In the 3-type view, freezing rain, ice pellets, and any mixing of two or more precipitation types is considered to be a wintry mix.

In the 4-type view, the displayed precipitation type in the case of mixes is generally the most impactful type of precipitation contained in the mix. For example, in a freezing rain-ice pellet mix, the ice pellets will accumulate on the ground and the freezing rain will glue the pellets together into a glaze ice coating. Additionally, only the freezing rain will accumulate on power lines. Since the freezing rain is probably producing a higher impact than the ice pellets in this situation, RadarView's 4-type display shows this as freezing rain.

Accessibility

The default color tables have been designed intentionally to be usable for both fully color-sighted users and most colorblind users. I've used the Pilestone Color Blindness Simulator to guide development of the color tables, and they should work for users with Deutanopia (the most common form of red-green colorblindness), Protanopia (a less common form of red-green colorblindness), and Tritanopia (blue-yellow colorblindness). I have not yet been able to make them usable for users with Monochromacy (black-and-white vision), but I hope to do that at some point in the future.

Though I've my best on the others, I have not checked the default color table for the 12-ptype precipitation depiction

The velocity color table is currently not very good for those with Deutanopia or Protanopia. I will hopefully be soon fixing or working around this

Products and How to Use Them

Base Reflectivity

More complete description coming later. For now:

Reflectivity is a measure of how much of a radar signal is reflected back to the radar site. Higher reflectivity values usually signify heavier precipitation, and low values (≤20 dBZ for rain, ≤5 for snow) usually signify no precipitation. Radar displays often use cool colors to show light/no precipitation and warm colors to show heavy precipitation. Colors used in RadarView by default are as follows:

Color Meaning (rain) Meaning (snow)
Black/Gray No rain No snow
Dark Blue No rain Very light snow
Blue-Green Very light rain Light snow
Green Light rain Moderate snow
Yellow Moderate rain Heavy snow
Red Heavy rain Extremely heavy snow
Magenta Extremely heavy rain, possibly hail -

Base Velocity

More complete description coming later. For now:

Doppler Velocity is a measure of how much the storm is moving toward or away from the radar site. Greens and Blues indicate motion toward the radar; Reds and Oranges indicate motion away.

Spectrum Width

More complete description coming later. For now:

Spectrum Width is a measure of how much variation there is in the wind within a single radar data pixel (often called a bin). It is equivalent to the standard deviation of velocity within the bin. Gray colors indicate low variation while red and green colors indicate significant variation.
This can be used to locate regions of convergence as well as the center of a Tornado Vortex Signature (TVS).

Correlation Correlation

More complete description coming later. For now:

Correlation Coefficient is a measure of how correlated the sizes of the objects within the radar's beam are. A correlation coefficient of 1.0 means that the objects are all the same size. A lower correlation coefficient indicates variation in sizes. Red colors indicate high correlation coefficients while yellow and blue colors indicate lower values.
It can be used in three ways.
The first way is to look for a region of lowered values in a ring centered on the radar site. These lowered values indicate a melting layer, which can be helpful in analyzing winter storms.
The second way is to look for a region of somewhat lowered values (roughly 0.8 to 0.9) located in the same place as very high reflectivity values (>60 dBZ). This indicates the possibility of large hail.
The third way is to look for a sharply defined region of very low values (<0.75) located in the same place as strong rotation in velocity and ideally also a hook shape on reflectivity. This indicates a Tornado Debris Signature (TDS), where debris from buildings has been lifted into the radar beam. It is usually visible when strong tornadoes are directly impacting towns.

Known Issues

  • Sometimes, radar data from the previous site is displayed in place of data from the currently selected site
  • Differential Reflectivity scans are all sorts of messed up
  • Map "wobbles" a bit when panning
  • Large-scale warnings in some areas don't render in the program.
  • Large-scale warnings make the program significantly slower to render
  • Velocity scans sometimes have large sections of null data, which disrupts the storm-relative velocity computation
  • Dealiasing algorithm does not entirely work right
